# Product Roadmap

1. [ ] Enhanced Attendance Tracking with Seat Time — Build comprehensive daily/weekly/monthly attendance tracking with precise seat time calculation, including check-in/check-out interfaces for coordinators and automated time aggregation to support credit calculations and program compliance reporting. `L`

2. [ ] Earned Credit System Foundation — Create earned credit tracking infrastructure supporting time-based credits linked to attendance records, including resident-facing views showing accumulated credits, credit transaction history, and progress toward sentence reduction milestones. `L`

3. [ ] Activity-Based Credit Support — Extend credit system to handle non-time-based activities (e.g., "1 knitted hat = 3 hours"), with admin interfaces for defining credit equivalencies, submission workflows for residents to log activities, and coordinator approval processes. `M`

4. [ ] Basic Report Export Functionality — Implement core reporting infrastructure with export capabilities (PDF/CSV) for attendance records, enrollment rosters, and program completion lists, with role-based report access and basic filtering options. `M`

5. [ ] Program Outcomes Dashboard — Build analytics dashboard showing quarterly and annual program performance metrics including completion rates, average seat time, credit earned, student progression through milestones, and dropout/risk indicators with drill-down capabilities. `L`

6. [ ] Facility Comparison Reporting — Create department administrator reporting tools for cross-facility benchmarking, comparing program effectiveness metrics, enrollment trends, completion rates, and earned credit distribution across facilities with visualization and export capabilities. `L`

7. [ ] Information Architecture Optimization — Redesign high-density screens to present information more digestibly using progressive disclosure, improved data visualization, card-based layouts, and customizable dashboard widgets to reduce cognitive overload while maintaining access to detailed data. `M`

8. [ ] Resident Progress Portal Enhancement — Build resident-first progress tracking interface highlighting achievements, earned credits, upcoming milestones, recommended content, and personalized learning pathways with visual progress indicators and motivational elements. `M`

> Notes
> - Order items by technical dependencies and product architecture
> - Each item should represent an end-to-end (frontend + backend) functional and testable feature
> - Priority focuses on reporting and tracking capabilities that demonstrate program value and support sentence reduction goals
> - Foundation features (items 1-3) enable later analytics and reporting features (items 4-6)
> - UX improvements (items 7-8) ensure data is actionable and uplifting for all users
> - All features maintain the "resident first" philosophy while meeting administrative requirements
